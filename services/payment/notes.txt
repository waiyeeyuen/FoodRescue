### Current End Points

1. Health Check
GET /payments/health

Expected response:
{
  "status": "ok",
  "service": "payment"
}

2. Create checkout session
POST /payments/checkout-session
Body Example:
{
  "orderId": "order_001",
  "userId": "user_001",
  "currency": "sgd",
  "items": [
    {
      "name": "Food Rescue Meal Box",
      "unitAmount": 500,
      "quantity": 2
    }
  ]
}

Expected response:
{
  "paymentId": "some-uuid",
  "status": "pending",
  "checkoutUrl": "https://checkout.stripe.com/..."  < THIS WILL BE THE LINK FOR USER TO PAY
}

3. Get Payment (1 only)
GET /payments/:paymentId

4. Get All Payment
GET /payments

4. Stripe webhook
POST /payments/webhook
(Called by stripe not me)



2. Folder Structure

payment/
│
├── .env
├── package.json
├── index.js
│
├── routes/
│   └── paymentRoutes.js
│
├── controllers/
│   └── paymentController.js
│
├── services/
│   ├── stripeService.js
│   ├── firebaseService.js
│   └── paymentRepository.js
│
└── utils/
    └── config.js

.env
stores port, Stripe secret key, webhook secret, frontend redirect URLs, firebaseservice key

package.json
stores dependencies and scripts

index.js
main entry point, starts Express server, mounts routes, defines webhook route

routes/paymentRoutes.js
declares payment-related API endpoints such as health check, create checkout session, 
refund payment, and get payment

controllers/paymentController.js
contains route logic such as creating Stripe checkout sessions, retrieving payments,
processing refunds, and handling Stripe webhook events

services/stripeService.js
contains Stripe SDK logic for checkout process and refund logic

services/firebaseService.js
initializes Firebase Admin SDK and connects the payment service to Firestore

services/paymentRepository.js
handles database operations for payments such as creating, retrieving, and updating payment records

utils/config.js
loads env variables using dotenv
